# Glossary

| Term                                                    | Definition                                                                                                                                                                                                                                                                                 |
|---------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 374a SGB V                                              | Legal paragraph in the German Social Code Book V concerning the integration of open and standardized interfaces in medical aids and implants.                                                                                                                                              |
| Access Token or `access_token`                          | OAuth token for accessing protected resources.                                                                                                                                                                                                                                             |
| Authorization Server                                    | OAuth server of the medical aid/implant that manages the authorization of DiGAs for data access.                                                                                                                                                                                           |
| Backend                                                 | Server-side application of a DiGA or medical aid/implant through which data transmission takes place.                                                                                                                                                                                      |
| BfArM                                                   | Federal Institute for Drugs and Medical Devices – responsible for establishing and publishing the electronic directory for interoperable interfaces of medical aids and implants, managing the DiGA directory, as well as the central terminology server for the German healthcare system. |
| CA/Browser Forum                                        | Trust space of the Internet PKI – certification authorities used for TLS connections.                                                                                                                                                                                                      |
| Certificate Transparency (CT)                           | Transparency framework for TLS certificates to detect unauthorized certificates.                                                                                                                                                                                                           |
| Client-ID or `client_id`                                | Unique OAuth client identifier of a DiGA, defined by BfArM.                                                                                                                                                                                                                                |
| CodeSystem                                              | FHIR coding system with unique codes for specific concepts (e.g., CS_OAuth_Scopes).                                                                                                                                                                                                        |
| Device                                                  | FHIR resource for transmitting metadata about measuring devices and device instances.                                                                                                                                                                                                      |
| DeviceDefinition                                        | FHIR resource with information on a specific medical aid backend and supported sensors/measuring devices.                                                                                                                                                                                  |
| DeviceMetric                                            | FHIR resource for transmitting measurement configuration and calibration information.                                                                                                                                                                                                      |
| DiGA                                                    | Digital Health Application – digital medical devices of low-risk classes that can support insured persons in the treatment of diseases or the compensation of impairments.                                                                                                                 |
| DiGA Manufacturer                                       | Manufacturer of digital health applications that distributes a DiGA listed in the DiGA directory.                                                                                                                                                                                          |
| DiGA Registry (DiGA VZ)                                 | Central information source for all certified, prescribable, and reimbursable DiGAs.                                                                                                                                                                                                        |
| Consent                                                 | Explicit agreement of the insured person to transmit data from the medical aid to a DiGA based on GDPR.                                                                                                                                                                                    |
| FHIR (Fast Healthcare Interoperability Resources)       | Standard for the exchange of healthcare data, used in version R4 for all healthcare data transmissions.                                                                                                                                                                                    |
| FQDN (Fully Qualified Domain Name)                      | Unique domain name for identifying DiGA and HiMi backends, as specified in the TLS certificate.                                                                                                                                                                                            |
| Frontend                                                | User interface (web apps, native apps, hybrid apps) of a DiGA or medical aid for interaction with the user.                                                                                                                                                                                |
| gematik                                                 | Agency for Telematics Applications of the Health Card – responsible for the technical specifications of the §374a interface.                                                                                                                                                               |
| GMI (Glucose Management Index)                          | Equivalent for the HbA1c laboratory value which is calculated from the average readings of a CGM system |
| HiMi (Medical Aids and Implants)                        | Tangible aids or technical products for medical care that are provided at the expense of statutory health insurance (GKV).                                                                                                                                                                 |
| HiMi Manufacturer                                       | Manufacturer of medical aids and implants falling under the legal regulation of §374a SGB V.                                                                                                                                                                                               |
| HiMi-SST-VZ (Medical Aid Interface Directory)           | Overview of all medical aids with an implemented §374a interface.                                                                                                                                                                                                                          |
| Interoperability                                        | Ability of different systems to communicate and exchange data in a standardized manner.                                                                                                                                                                                                    |
| KIG                                                     | Competence Center for Interoperability in Healthcare at gematik – legally mandated to define technical specifications according to §374a SGB V.                                                                                                                                            |
| LOINC (Logical Observation Identifiers Names and Codes) | Coding system for laboratory values and vital signs.                                                                                                                                                                                                                                       |
| Monitoring                                              | Self-logging of DiGA and HiMi to document the functionality of the §374a interface.                                                                                                                                                                                                        |
| mTLS (Mutual Transport Layer Security)                  | Mutually authenticated TLS channel between DiGA and HiMi backend.                                                                                                                                                                                                                          |
| OAuth 2.0 (Open Authorization)                          | Standard for securely authorizing a DiGA to access medical aid/implant data.                                                                                                                                                                                                               |
| Observation                                             | FHIR resource for transmitting measurement values of exactly one vital sign/vibW (e.g., blood glucose, blood pressure).                                                                                                                                                                    |
| Pairing                                                 | Linking process between DiGA and medical aid – process of connecting systems with user consent.                                                                                                                                                                                            |
| Pairing ID                                              | Pairing-specific user pseudonym for uniquely assigning consent between DiGA and medical aid.                                                                                                                                                                                               |
| Profile                                                 | Set of contraints that must be considered when using a specific FHIR resource type in a specific context (e.g. § 374a SG V). |
| Redirect-URI or `redirect_uri`                          | Redirect URI to which a HiMi sends the user after issuing the authorization code.                                                                                                                                                                                                          |
| Refresh Token                                           | OAuth token to renew the access token without user interaction.                                                                                                                                                                                                                            |
| Resource Server                                         | FHIR server of the medical aid/implant providing the protected healthcare data.                                                                                                                                                                                                            |
| Scope                                                   | OAuth permission defining which data a DiGA may access (e.g., vibW-specific scopes).                                                                                                                                                                                                       |
| SMART 2.0                                               | Standard for fine-grained access control with patient-specific scopes and finer-grained resource constraints.                                                                                                                                                                              |
| %TIR (Times in Range) | Percentage of time where a patient's glucose value is in the normal range (70-180 mg/dl)  |
| UCUM (Unified Code for Units of Measure)                | Unit system for measurement values in medical applications.                                                                                                                                                                                                                                |
| ValueSet                                                | FHIR value list with defined codes for a specific context (e.g., VS_BloodGlucose).                                                                                                                                                                                                         |
| MIV (Mandatory Interoperable Value)                   | Central granularity level for standardized data transmission (e.g., blood glucose, blood pressure).                                                                                                                                                                                        |
| ZTS (Central Terminology Server)                        | Central terminology server for the German healthcare system – provides binding coding systems and value lists.                                                                                                                                                                             |
