Components and Roles

Component Role / Responsible Functions / Data in Context of § 374a SGB V Interfaces / APIs
DiGA Directory (DiGA-VZ) BfArM - Lists DiGA authorized to use § 374a interface
- Provides authenticity verification of DiGA
- Provides legally permissible authorizations
- Provides registration info for DiGA at device
- Optional: notification mechanism for entry changes		 None
HiMi Directory Interface (HiMi-SST-VZ) BfArM (in development) - Lists devices/implants with implemented § 374a interface
- Provides manufacturer/device info
- Provides supported sensors/measuring devices
- Provides data made available via interface
- Authenticity verification of devices
- Endpoint discovery
- Optional: notification mechanism for entry changes		 None (working title: HiMi-SST-VZ API)
Zertraler Terminologieserver (ZTS) BfArM - Provides binding coding systems & value sets
- Versioning of value sets
- Signals changes to value sets		 None
Medical Device / Implant – Backend Device/Implant Manufacturer - Requests & manages user consent
- Regular validity checks
- Allows revocation/amendment of consent
- Authenticates DiGA
- Authorizes DiGA based on pseudonym
- Provides medical data as FHIR resources per ZTS value sets
- Monitoring & documentation of interface functionality		 - RESTful FHIR API
- OAuth 2.0 ACG API
- Frontend API (manufacturer-specific)
- ZTS API
- (optional: DiGA API)
Medical Device / Implant – Frontend Device/Implant Manufacturer - Presents requested/authorized permissions
- Captures user consent
- Presents granted consents
- Captures revocation/amendments
- Informs about changes in consent/coupling		 - Frontend API (manufacturer-specific)
Consent: accessible and informed consent
DiGA – Backend DiGA Manufacturer - Initiates authorization
- Requests & manages consent
- Regular validity checks
- Allows revocation/amendment
- Authenticates device/implant
- Retrieves data (FHIR)
- Interprets data per ZTS value sets
- Monitoring & documentation		 - RESTful FHIR API
- OAuth 2.0 ACG API
- Frontend API (manufacturer-specific)
- ZTS API
- (optional: HiMi-SST-VZ API)
DiGA – Frontend DiGA Manufacturer - Displays § 374a interface info
- Displays consent info
- Presents supported/authorized devices & sensors
- Presents required permissions
- Captures user selection & consent
- Presents granted consents
- Captures revocation/amendment		 - Frontend API (manufacturer-specific)
- /authorize endpoint of OAuth ACG API
Consent: accessible and informed consent
Security: RFC 8252
